NAME
    gemini-cli - Bash terminal interface for the Gemini API

SYNOPSIS
    gemini-cli
    gemini-cli <session_name>
    gemini-cli --resume <session_name>
    gemini-cli --list
    gemini-cli --models
    gemini-cli --test-model <model_id>
    gemini-cli --test-model <model_id> --save
    gemini-cli --select-model
    gemini-cli --select-model --save
    gemini-cli --set-api-key
    gemini-cli --version
    gemini-cli --help
    gemini-cli -h

DESCRIPTION
    gemini-cli is a lightweight Bash-based terminal chat interface for
    the Google Gemini API.

    It is the minimal, line-oriented edition of the gemini-terminal-tools
    project. It is intended to remain scriptable, readable, and
    dependency-light.

    It provides:
        - persistent chat sessions
        - named and resumable conversations
        - timestamped logs
        - rolling conversation context
        - resume-time context rebuild from session logs
        - context snapshots stored in session logs
        - temporary Project Lead startup notice for new blank sessions
        - config file support
        - automatic code attachment extraction
        - model listing
        - user-operated model testing
        - numbered model selection
        - API-key setup helper

USAGE
    First-time setup:

        gemini-cli --set-api-key
        gemini-cli --models
        gemini-cli --select-model

    Start a normal timestamped chat:

        gemini-cli

    Start a named chat:

        gemini-cli android-backup

    Resume a named chat:

        gemini-cli --resume android-backup

    List saved chats:

        gemini-cli --list

    List currently available Gemini models:

        gemini-cli --models

    Test one model:

        gemini-cli --test-model gemini-2.5-flash

    Test one model and save it if successful:

        gemini-cli --test-model gemini-2.5-flash --save

    Choose a model manually:

        gemini-cli --select-model

SESSION MANAGEMENT
    gemini-cli
        Start a new timestamped session.

    gemini-cli <name>
        Start or open a named session. If the named log already exists,
        gemini-cli rebuilds recent conversation context from it.

    gemini-cli --resume <name>
        Resume an existing named session. If the session does not exist,
        the command exits with an error.

        On resume, gemini-cli rebuilds recent conversation context from
        the session log so follow-up questions can refer to earlier messages.

    gemini-cli --list
        List saved sessions.

INPUT
    The Bash version uses a multiline Ctrl+D send workflow.

    Type or paste input, then press Ctrl+D to submit it.

    Ctrl+C exits the running chat.

STARTUP NOTICE
    In a new blank session, gemini-cli can display a temporary transcript-style
    Project Lead notice from remfan1994.

    Notice text:

        I strongly encourage everyone to get cruetly-free VEGETARIAN food and
        remember the 'bloodguilt' curse from the Bible.  -Project Lead,
        remfan1994

    Behavior:
        - shown only in new blank sessions
        - removed from the visible terminal when the first message is sent
        - not written to the session log
        - not sent to Gemini
        - not stored in conversation memory

    Config key:

        STARTUP_NOTICE=1

    Disable with:

        STARTUP_NOTICE=0

MODEL DISCOVERY
    Available Gemini models can be listed with:

        gemini-cli --models

    They can also be listed directly with curl:

        curl "https://generativelanguage.googleapis.com/v1beta/models?key=YOUR_API_KEY"

    gemini-cli filters the returned list to models that report
    generateContent support.

MODEL TESTING
    Autoscan has been removed from the Bash version.

    Instead of probing every available model, gemini-cli tests only a
    user-selected model.

    From the shell:

        gemini-cli --test-model <model_id>

    Example:

        gemini-cli --test-model gemini-2.5-flash

    The tester sends MODEL_TEST_PROMPT to the selected model and prints the
    raw response. The user reads the response and decides whether the model is
    useful.

    To save a model after a successful test:

        gemini-cli --test-model <model_id> --save

    The removed autoscan command:

        gemini-cli --autoscan-model

    now exits with an explanation and points users toward --models,
    --test-model, and --select-model.

MODEL SELECTION
    A numbered model selection menu is available:

        gemini-cli --select-model

    The tool prints available generateContent models, asks the user to choose
    a number, offers to test the selected model, then offers to save the chosen
    MODEL value to the config file.

    To save without a second confirmation prompt:

        gemini-cli --select-model --save

MODEL FALLBACK
    If MODEL is not configured, gemini-cli uses:

        gemini-2.5-flash

    as a fallback model.

    This is not a permanent recommendation and not a guarantee that the model
    will always be available. Users should prefer:

        gemini-cli --models
        gemini-cli --test-model <model_id>
        gemini-cli --select-model

    to choose a currently available model.

API KEY SETUP
    API key may be supplied by environment variable:

        export GEMINI_API_KEY=your_key

    It may also be placed in the config file:

        GEMINI_API_KEY=your_key

    gemini-cli can prompt for and save the key:

        gemini-cli --set-api-key

    Saving the API key stores it in plaintext. The script attempts to chmod
    the config file to 0600.

CONFIGURATION
    Configuration file:

        ~/.config/gemini-cli/config

    Format:

        KEY=VALUE

    Supported keys:

        MODEL
            Gemini model identifier.

        CONTEXT_TURNS
            Number of recent user/assistant exchanges sent back to Gemini as
            conversation context.

            Default: 12

        HISTORY_LIMIT
            Legacy alias for CONTEXT_TURNS. Kept for compatibility with older
            config files.

        GEMINI_API_KEY
            Optional API key. Environment variable takes priority.

        SESSION_DIR
            Directory used for session log storage.
            Default: ~/.gpt

        ATTACHMENT_DIR
            Directory used for extracted code artifacts.
            Default: SESSION_DIR/attachments

        MODEL_TEST_PROMPT
            Prompt used by --test-model and --select-model model testing.

        STARTUP_NOTICE
            Controls whether the temporary remfan1994 Project Lead notice is
            displayed in new blank sessions.

            Values:
                1, yes, y, true, on
                    enabled

                0, no, false, off
                    disabled

ENVIRONMENT
    GEMINI_API_KEY
        Preferred API key source.

        Precedence:
            1. Environment variable
            2. Config file

FILES
    ~/.config/gemini-cli/config
        Configuration file.

    SESSION_DIR/*.log
        Session log files.

    ATTACHMENT_DIR/
        Extracted code block attachments.

ATTACHMENT SYSTEM
    Markdown fenced code blocks returned by Gemini are extracted from
    terminal display and saved into ATTACHMENT_DIR.

    The original raw Gemini response remains preserved in the session log.

    File names follow this pattern:

        <session>-<type>-NN.<extension>

    Examples:

        android-backup-python-01.py
        linux-help-bash-02.sh
        notes-json-01.json

SESSION NAMES
    Session names are sanitized for filesystem safety.

    Characters outside letters, numbers, dot, dash, and underscore are
    converted to underscores.

MEMORY MODEL
    gemini-cli keeps a rolling conversation context in memory.

    CONTEXT_TURNS controls how many recent user/assistant exchanges are sent
    back to Gemini with each request.

    When resuming an existing session, gemini-cli rebuilds recent context from
    the session log.

    The Bash version now stores explicit context snapshots in the session log.
    When rebuilding context, it prefers the latest saved snapshot and falls
    back to recent chat entries if no snapshot exists.

    The session log remains the permanent archive. The context buffer is only
    the recent window sent back to Gemini.

LIMITATIONS
    - Requires Bash
    - Requires curl
    - Requires python3
    - Requires internet access
    - API quota restrictions apply
    - Basic terminal input only; no ncurses-style editing
    - Ctrl+D is used to send multiline input
    - The Project Lead startup notice can only be removed visually by clearing
      the terminal screen after the first message is sent
    - --test-model verifies only one selected model and does not guarantee
      overall model quality

VERSION
    gemini-cli version 2.5
